# SRAM 尺寸优化算法使用说明

本文档介绍 `size_optimization/` 目录下的优化算法框架，包括参数空间定义、目标函数、算法选择和实验管理。

---

## 1. 功能说明

本框架以 SRAM 电路仿真为黑盒，对以下变量进行联合自动寻优：

**架构参数（离散）**

| 变量 | 候选值 |
|------|-------|
| 行数 | {16, 32, 64, 128, 256, 512} |
| 列数 | {16, 32, 64, 128, 256, 512} |
| 阵列数 | 由 行数 × 列数 × 阵列数 = 总容量 自动确定 |

Column Mux 当前固定为 2 路复用（`choose_columnmux=True` 时 `mux_in=2`，关闭时为 1），不作为搜索变量。

**Bitcell 参数（混合）**

| 变量 | 类型 | 范围 |
|------|------|------|
| pull-down 宽度 | 连续 | [45, 300] nm |
| pull-up 宽度 | 连续 | [45, 300] nm |
| pass-gate 宽度 | 连续 | [45, 300] nm |
| 沟道长度 | 连续 | [40, 80] nm |
| NMOS Vt 类型 | 离散 | {VTL, VTG, VTH} |
| PMOS Vt 类型 | 离散 | {VTL, VTG, VTH} |

默认总容量为 32 KB = 262,144 bit，阵列数据此自动推导。

## 2. 两阶段优化策略

`experiment.py` 实现了架构-晶体管联合两阶段优化：

- **Stage 1（架构搜索）**：在容量约束下搜索最优行列数配置，目标是找到兼顾延迟、功耗、面积的中等规模配置。
- **Stage 2（晶体管定尺寸）**：以 Stage 1 最优架构为基础，搜索 pull-down/pull-up/pass-gate 宽度、沟道长度及 Vt 类型。

两阶段不可互相替代，单独优化架构或单独优化晶体管都不如联合运行效果好。

## 3. 目标函数

### 3.1 单目标 FoM（SA / PSO / CBO / SMAC / RoSE-Opt）

$$\text{FoM} = \log_{10}\!\left(\frac{\text{SNM}_\text{min}}{P_\text{max} \times D_\text{max} \times \sqrt{\text{Area}}}\right)$$

其中 `D_max = max(读延迟, 写延迟)`，`P_max = max(读功耗, 写功耗)`，均含架构惩罚修正值。FoM 越大表示综合性能越好（低功耗、小面积、短延迟、大噪声裕度同时优化）。

### 3.2 多目标（NSGA-II / MOEA/D）

直接以 SNM、Power、Delay、Area 四个原始指标为优化目标，输出完整 Pareto 前沿，无需标量化聚合。

### 3.3 面积估算

面积由 `utils.py` 中的 `estimate_bitcell_area()` / `estimate_total_macro_area()` 自动计算，无需 SPICE 仿真。

**Bitcell 面积**（基于 FreePDK45 布局规则估算）：

```
cell_width  = max(2×W_PD + W_PG, 2×W_PU + W_PG) + 间距
cell_height = 2×L + 多晶间距 + 扩散间距
bitcell_area = cell_width × cell_height
```

**阵列总面积**（含外围电路）：

```
array_area   = num_rows × num_cols × bitcell_area
macro_area   = array_area × 外围开销系数（约 1.1~1.2）
total_area   = macro_area × n_arrays
```

其中 `n_arrays` 由总容量约束自动推导（`n_arrays = capacity / (num_rows × num_cols)`）。多阵列情况下面积按线性叠加，外围逻辑（decoder、sense amp 等）随阵列数增加而增加，是优化中面积项的主要来源。

FoM 中的 `Area` 使用 `total_area`（单位 m²），因此缩小 `n_arrays`（即选择较大的单个阵列）通常会同时降低面积和 FoM 分母，是架构优化的核心收益。

conda activate openyield

## 6. 快速上手

各 demo 脚本在项目根目录直接运行即可，随机种子写在脚本 `__main__` 中（如需更改可直接编辑脚本顶部的 `SEED`）。

无参数脚本（SA / PSO / CBO / CMA-ES / tSS-BO）：

```bash
cd /path/to/OpenYield
python size_optimization/demo_sa.py
python size_optimization/demo_pso.py
python size_optimization/demo_cbo.py
python size_optimization/demo_cmaes.py
python size_optimization/demo_tssbo.py
```

多目标脚本（MOBO / NSGA-II / MOEAD）支持命令行参数 `--max_iter` 和 `--circuit_mode`（1=真实电路，2=等效电路），MOEAD 另有 `--h`（权重向量划分数）：

```bash
python size_optimization/demo_mobo.py   --max_iter 50  --circuit_mode 1
python size_optimization/demo_nsgaii.py --max_iter 800 --circuit_mode 1
python size_optimization/demo_moead.py  --max_iter 50  --circuit_mode 1 --h 3
```

结果保存在 `size_optimization/experiment/` 下对应的实验目录中。

## 7. 两阶段 / 联合优化入口（`experiment.py`）

`experiment.py` 为交互式入口，直接运行后按提示依次输入选项即可，无需命令行参数：

```bash
cd /path/to/OpenYield
python size_optimization/experiment.py
```

运行后依次询问：

1. **模式**：`1=联合优化（joint）`，`2=两阶段固定架构（two-stage-fixed-configs）`（默认 1）
2. **算法**：联合模式支持 SA / PSO / SMAC / CBO / RoSE_Opt / MOEAD / MOBO / NSGA-II / tSS-BO / CMA-ES / CPN（默认 SA）
3. **迭代次数**：联合模式默认 40，两阶段模式默认 400
4. **种子**：base seed（默认 42）及种子数量（默认 1，可一次跑多个种子）
5. **电路模式**：`1=真实电路`，`2=等效电路`（默认 1）

两阶段固定架构模式（mode=2）会在预设的 5 组行列配置（128×32、64×32、32×32、32×16、16×16）中选择一组或全部跑 Stage 2。

## 8. 参数空间配置（`config_sram.yaml`）

```yaml
parameters:
  pd_width:   {lower: 1.35e-7, upper: 4.05e-7}   # pull-down 宽度
  pg_width:   {lower: 9.0e-8,  upper: 2.7e-7}    # pass-gate 宽度
  pu_width:   {lower: 9.0e-8,  upper: 2.7e-7}    # pull-up 宽度
  length:     {lower: 5.0e-8,  upper: 1.0e-7}    # 沟道长度
  nmos_model: {choices: [NMOS_VTL, NMOS_VTG, NMOS_VTH]}
  pmos_model: {choices: [PMOS_VTL, PMOS_VTG, PMOS_VTH]}
```

## 9. 结果文件说明

`experiment.py` 的运行结果写入 `size_optimization/experiment/<算法名>/` 目录，主要文件：

| 文件 | 内容 |
|------|------|
| `stage1_iterations_live.csv` | Stage 1（架构搜索）每次评估的参数和结果，实时追加 |
| `stage2_iterations_live_<label>.csv` | Stage 2（晶体管定尺寸）每次评估记录 |
| `joint_iterations_live_joint_<tag>.csv` | 联合优化模式的迭代记录 |
| `iterations_<label>_<算法>_<时间戳>.csv` | 单次运行的完整迭代汇总 |

Pareto / 收敛相关的图保存在 `size_optimization/new_result/`（如 `power_delay_pareto_stage1.png`、`power_delay_pareto_stage2_<label>.png`）。

各 `demo_*.py` 脚本默认将关键指标打印到终端（最优 FoM、SNM、延迟、功耗、面积），不强制落盘；如需保存可在脚本中调用 `exp_utils` 提供的 `save_best_result` / `save_pareto_front` / `plot_merit_history` 等辅助函数。

读取迭代 CSV 示例：

```python
import pandas as pd
df = pd.read_csv("size_optimization/experiment/SA/stage1_iterations_live.csv")
print(df.describe())
```

## 10. 算法实验建议

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 快速验证 / 基线 | `demo_random.py` / `demo_sa.py` | 无额外依赖，收敛稳定 |
| 样本效率优先（评估代价高） | `demo_cbo.py` / `demo_tssbo.py` | 贝叶斯优化少样本下表现好 |
| 最优单一 FoM | `demo_moead.py` | 多目标分解，综合性能优 |
| 多目标权衡分析 | `demo_mobo.py` / `demo_nsgaii.py` | 输出完整 Pareto 前沿 |
| 架构 + 晶体管联合 | `experiment.py` 两阶段模式 | 架构搜索贡献主要 FoM 增益 |
